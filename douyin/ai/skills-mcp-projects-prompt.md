# Skills、MCP、Projects、Prompt 都在说什么

> 清华姜学长这个视频试图厘清 AI 领域中几个常被混淆的概念边界：**skills（能力定义）**、**mcp（协议标准）**、**projects（项目框架）**、**prompt（交互语言）**——它们分别解决不同层级的问题，却在实际应用中被混为一谈。

---

## 概念图谱

```
AI Agent 生态架构
│
├─ 交互层 ────── Prompt（提示词）
│               └─ 人机对话的语言接口
│
├─ 能力层 ────── Skills（技能）
│               └─ Agent 能做什么（功能定义）
│
├─ 协议层 ────── MCP（Model Context Protocol）
│               └─ 模型与外部系统的通信标准
│
└─ 组织层 ────── Projects（项目）
                └─ 任务/目标的结构化封装
```

---

## 四大概念详解

| 概念 | 本质 | 解决的问题 | 类比 |
|------|------|-----------|------|
| **Prompt** | 输入指令 | 如何让 AI 理解你的需求 | 对下属说话的方式 |
| **Skills** | 能力清单 | 定义 Agent 具备的功能 | 员工的岗位说明书 |
| **MCP** | 通信协议 | 模型如何调用外部工具 | USB-C 接口标准 |
| **Projects** | 项目容器 | 组织多步骤复杂任务 | 一个完整的项目文件夹 |

### 1. Prompt（提示词）
- **定位**：人机交互的「语言层」
- **核心问题**：用什么样的表达方式，让大模型输出你想要的结果
- **进阶方向**：
  - Zero-shot / Few-shot 提示
  - Chain-of-Thought 思维链
  - ReAct 推理+行动框架

### 2. Skills（技能）
- **定位**：Agent 的能力「功能层」
- **核心问题**：这个 Agent 被允许/能够执行哪些操作
- **常见形式**：
  - Function Calling 函数定义
  - Tool Use 工具调用
  - API 集成能力

### 3. MCP（Model Context Protocol）
- **定位**：标准化「协议层」
- **核心问题**：不同模型、不同工具之间如何统一通信
- **背景**：Anthropic 提出的开放协议，旨在解决模型与外部数据源/工具的互操作问题
- **类比**：就像 HTTP 协议统一了网页通信，MCP 试图统一 AI 与世界的交互

### 4. Projects（项目）
- **定位**：任务组织「结构层」
- **核心问题**：如何将复杂目标拆解、执行、追踪
- **典型场景**：
  - 多步骤工作流
  - 长期记忆管理
  - 协作任务分配

---

## 关键洞察

> **混淆的根源**：这四个概念在实际使用中常常是「你中有我」的
> 
> **区分的价值**：厘清层级后，可以更准确地讨论「我在优化什么」
