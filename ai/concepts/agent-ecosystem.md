# Skills、MCP、Projects、Prompt 辨析

> **来源**：抖音 @清华姜学长  
> 厘清 AI 领域中几个常被混淆的概念边界

---

## 概念图谱

```
AI Agent 生态架构
│
├─ 交互层 ────── Prompt（提示词）
│               └─ 人机对话的语言接口
│
├─ 能力层 ────── Skills（技能）
│               └─ Agent 能做什么（功能定义）
│
├─ 协议层 ────── MCP（Model Context Protocol）
│               └─ 模型与外部系统的通信标准
│
└─ 组织层 ────── Projects（项目）
                └─ 任务/目标的结构化封装
```

---

## 四大概念详解

| 概念 | 本质 | 解决的问题 | 类比 |
|------|------|-----------|------|
| **Prompt** | 输入指令 | 如何让 AI 理解你的需求 | 对下属说话的方式 |
| **Skills** | 能力清单 | 定义 Agent 具备的功能 | 员工的岗位说明书 |
| **MCP** | 通信协议 | 模型如何调用外部工具 | USB-C 接口标准 |
| **Projects** | 项目容器 | 组织多步骤复杂任务 | 一个完整的项目文件夹 |

### Prompt
- **定位**：人机交互的「语言层」
- **进阶**：Zero-shot / Few-shot / CoT / ReAct

### Skills
- **定位**：Agent 的能力「功能层」
- **形式**：Function Calling / Tool Use / API 集成

### MCP
- **定位**：标准化「协议层」
- **背景**：Anthropic 提出的开放协议
- **目标**：统一 AI 与世界的交互（类比 HTTP）

### Projects
- **定位**：任务组织「结构层」
- **场景**：多步骤工作流、长期记忆、协作分配

---

## 关键洞察

混淆的根源是四者在实际使用中「你中有我」。厘清层级后，可以更准确地讨论「我在优化什么」。
